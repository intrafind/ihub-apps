**Date**: 2026-02-19

## UI Design Changes

### Preview Badge Styling

The preview features are marked with a **blue badge** to distinguish them from regular features.

**Badge Specifications:**
- Background: `bg-blue-100` (light blue)
- Text: `text-blue-800` (dark blue)
- Text: "Preview" (English) / "Vorschau" (German)
- Rounded corners with padding

**Previous "Experimental" Badge:**
- Background: `bg-amber-100` (amber/yellow)
- Text: `text-amber-800` (dark amber)
- Text: "Experimental"

## Key Visual Changes

1. **Preview Category Order**: The "Preview" category appears **first** (order: 1)
2. **Preview Badge**: Blue badge with "Preview" text next to feature name
3. **Category Grouping**: Features are clearly grouped by category with headers
4. **Toggle Switches**: Standard toggle switches for enabling/disabling features

## Responsive Design

The features page is fully responsive:
- Mobile: Features stack vertically with full width
- Tablet: Features maintain card layout with adequate spacing
- Desktop: Full-width layout with clear category sections

## Accessibility

- Toggle switches have proper ARIA labels
- Color contrast meets WCAG AA standards
- Keyboard navigation fully supported
- Screen reader friendly with descriptive labels

## Translation Support

All feature names, descriptions, and UI text support multiple languages:
- English (`en`)
- German (`de`)

## Implementation Details

The preview badge is conditionally rendered based on the `preview` flag:

```jsx
{feature.preview && (
  <span className="inline-flex items-center px-2 py-0.5 rounded text-xs font-medium bg-blue-100 text-blue-800">
    {t('admin.features.preview', 'Preview')}
  </span>
)}
```

## Feature Categories Display Order

1. **Preview** (order: 1) - Contains preview features
2. **AI Capabilities** (order: 2) - AI-related features
3. **Content** (order: 3) - Content management features
4. **Analytics** (order: 4) - Analytics and tracking features
